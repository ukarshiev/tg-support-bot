# Базовые правила агентов

Главная точка входа — `AGENTS.md` в корне проекта. Этот файл — короткая каскадная памятка для агентов, которые читают `docs/agents/rules/**`.

Приоритет правил:

```text
system/developer > AGENTS.md > docs/agents/rules/** > старые локальные правила
```

## Расширенная выжимка принципа

Главный принцип — **не навреди**.

Агент не должен угадывать. Сначала факты: git, файлы, Docker, логи, миграции, настройки, версия/commit.

Если фактов не хватает — сказать Владыке, что нужно проверить, затем провести анализ и тесты.

При ошибках:

- искать корень проблемы;
- не закрывать симптом кучей заплаток;
- проверять официальную документацию, bug tracker, release notes и внешние источники;
- исправлять без лишней сложности;
- проверять соседние сценарии.

При доработках:

- оценивать плюсы и минусы;
- искать Best Practice;
- учитывать зависимости проекта;
- не срезать углы;
- не добавлять избыточность;
- явно называть риски новых библиотек или компонентов.

## Адаптация под tg-support-bot

- Пользователь — **Владыка**.
- Язык общения — русский.
- Официальная документация от разработчика: `https://docs.tg-support-bot.ru/`.
- Для библиотек и API использовать официальные docs и Context7, если он доступен.
- PostEditBot проверять только через API, прямой доступ к БД PostEditBot запрещён.
- Нашу интеграцию держать отдельно: `app/Modules/PostEditBotBridge/**`, `docker-compose.relaxa.yml`, отдельные настройки/миграции.
- Нельзя говорить “запушено”, “задеплоено”, “проверено на бою”, “CI прошёл” без фактической проверки.
